# Competitive Context: Stripe & Adyen

## Competitive Summary

| Dimension | JPMorgan MS | Stripe | Adyen |
|-----------|-------------|--------|-------|
| Revenue | ~$1.7B net | ~$5B+ | ~$2B+ |
| EBITDA Margin | ~Breakeven | Growing | ~50% |
| Primary Buyer | Enterprise, regulated, global | Developer-first, SMB→enterprise | Enterprise, global, omnichannel |
| Core Advantage | Trust, banking relationship, healthcare | DX, speed, ecosystem | Unified platform, margins, data |
| API Quality | Lagging | Best-in-class | Strong |
| Hosted UI | Weak | Excellent (Elements, Links) | Good |
| Agentic Commerce | No strategy | Leading (LLM toolkits) | Emerging |
| Healthcare/IIAS | Genuine differentiator | Minimal | Minimal |
| Debit Routing | Full PIN network access | Limited | Limited |

---

## Stripe — Deep Profile

### Strengths
- **Developer experience** — API design is the gold standard; documentation, SDKs, sandbox, error messages all best-in-class
- **Hosted UI** — Stripe Elements and Payment Links convert better than anything JPM offers
- **Ecosystem** — Stripe Billing, Radar, Connect, Issuing, Treasury create lock-in
- **Speed of innovation** — Ships new payment methods, features, and geographies faster than any competitor
- **LLM-native** — Building explicitly for agentic commerce (AI agents initiating payments)
- **Stripe Tax / Sigma** — Analytics and compliance as VAS

### Weaknesses
- **Enterprise trust** — Some large enterprises prefer bank-backed acquirer for counterparty risk
- **Healthcare** — No IIAS/FSA/HSA-native capability
- **Balance sheet** — Not a bank; can't offer same credit facilities, FX, or treasury products
- **Debit routing** — No PIN network direct access

### JPM's Response to Stripe
- Improve API documentation and sandbox quality
- Build hosted checkout parity
- Lead with trust + healthcare where Stripe is absent
- Target enterprise where total relationship (banking + payments) matters

---

## Adyen — Deep Profile

### Strengths
- **Unified platform** — Single platform for online, in-store, mobile; JPM operates separate stacks
- **EBITDA margin** — ~50% proves the model; superior cost discipline
- **Data** — Unified transaction data across channels powers fraud and insights
- **Merchant of record relationships** — Deep enterprise penetration (Spotify, McDonald's, Microsoft)
- **Issuer processing** — Adyen now has issuing capability too (similar to JPM's dual position)

### Weaknesses
- **US market share** — Historically weaker in US than Europe
- **SMB / developer** — Not a natural fit below enterprise tier
- **Banking products** — Not a bank; no FX, credit, treasury relationship

### JPM's Response to Adyen
- Match on omnichannel narrative (JPM has both online and POS acquiring)
- Use healthcare and regulated verticals as wedge
- Margin improvement story: if JPM reaches even 20% EBITDA margin, the strategic narrative changes

---

## Competitive Positioning for JPM

### Where JPM Wins Today
1. **Healthcare / FSA / HSA acquiring** — IIAS capability is a genuine moat
2. **Regulated enterprise** — Banking relationship, compliance posture, counterparty trust
3. **Debit routing** — Full PIN network access (STAR, NYCE, PULSE, ACCEL)
4. **International multinationals** — Full banking + acquiring relationship

### Where JPM Must Close the Gap
1. **Developer experience** — API docs, sandbox, SDK quality
2. **Hosted UI** — Checkout.js needs a complete rethink
3. **Payment method breadth** — BNPL, wallets, local payment methods
4. **Agentic / AI commerce** — No current strategy
5. **Time to value** — Onboarding and integration speed

---

## Messaging Principles

- Never lead with "we're cheaper" — race to the bottom
- Lead with: trust + healthcare + debit routing + banking relationship
- Acknowledge DX gap honestly; lead with roadmap
- Use Adyen margin story to justify platform investment internally
